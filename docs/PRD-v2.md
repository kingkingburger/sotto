# Sotto v2 — Product Requirements Document

> 작성일: 2026-03-09
> 상태: 확정 (유저 시나리오 인터뷰 완료)

---

## 1. 제품 비전

**"주간 도시락 메뉴 추천 + 실시간 재료 가격 모니터링"**

직장인 도시락족이 앱을 열면 바로 이번 주 메뉴를 받고, 각 레시피의 재료비를 실시간 시세로 확인할 수 있는 웹앱.

### 핵심 가치 (우선순위)
1. **레시피 중심 메뉴 추천** — 앱 오픈 즉시 주간 메뉴, 카드 탭하면 레시피 상세
2. **실시간 가격 모니터링** — 재료별 시세, 총 예상 비용, 가격 트렌드
3. **장보기 목록** — 보조 도구 (체크리스트 + 가격 표시)

### 핵심 유저
- **직장인 도시락족**: 일요일 저녁에 다음 주 도시락 메뉴를 정하는 사람
- 메뉴 고민 해소 + 재료비 파악이 동시에 필요

## 2. 디자인 원칙 (최우선)

> **"누가 봐도 우와 하는 디자인"**

| 원칙 | 설명 |
|------|------|
| **현대적 & 감각적** | Apple HIG 수준의 미니멀하면서도 세련된 UI |
| **부드러운 애니메이션** | 모든 전환에 spring animation, 카드 인터랙션에 micro-interaction |
| **진취적 디자인** | 대담한 타이포그래피, 그래디언트, 글래스모피즘 등 트렌디한 요소 |
| **모바일 & 데스크톱 동등** | 반응형이 아닌, 각 환경에 최적화된 레이아웃 |

### 디자인 참고
- Apple의 최신 디자인 언어 (Dynamic Island, visionOS 감성)
- Framer Motion / Spring animation
- 필터 UI: Apple 스타일 (구체적 형태는 디자인 단계에서 결정)

## 3. 유저 플로우 (확정)

### 변경사항 (v1 → v2)
| v1 | v2 |
|----|-----|
| 태그 선택 페이지 (/select) 존재 | **제거** — 필터로 대체 |
| 장보기 목록이 주요 기능 | **보조 도구**로 격하 |
| 하드코딩 가격 사전 | **실시간 가격 모니터링** |
| 매번 새로운 메뉴 | **이전 메뉴 유지** + 새로고침/하루 지나면 리셋 |
| YouTube 임베드 | **YouTube 앱으로 외부 이동** (링크) |

### 플로우

```
앱 오픈
  ↓
[메인] 주간 메뉴 그리드 (5일 기본)
  │  - 랜덤 추천 (기본)
  │  - 필터로 조정 가능 (예산/맛/양/간편/영양)
  │  - 개별 재뽑기 🎲 + 전체 새로고침 🔄
  │  - 7일로 변경 가능
  │  - 이번 주 예상 재료비 + 가격 트렌드
  │  - 메뉴는 유지됨 (새로고침 or 하루 지나면 리셋)
  │
  ├── 카드 탭 → [레시피 상세]
  │     - Hero 이미지
  │     - 재료 (실시간 가격 표시)
  │     - 영양정보
  │     - 조리순서
  │     - YouTube 영상 → 외부 링크
  │     - 재뽑기 가능
  │
  └── 메뉴 확정 → [장보기 목록] (보조)
        - 카테고리별 체크리스트
        - 재료별 실시간 가격
        - 총 예상 비용
        - 목록 복사
```

## 4. 화면별 상세 스펙

### 4.1 메인 화면 (주간 메뉴)

**진입**: 앱 오픈 즉시 (별도 선택 과정 없음)

**메뉴 카드 (간결)**:
- 레시피 이미지
- 음식명
- 예상 가격 (실시간)
- 조리시간
- 난이도
- YouTube 영상 유무 아이콘
- 칼로리
- 재뽑기 🎲 버튼

**상단 액션**:
- 필터 (기본: 없음 = 랜덤, 필요 시 태그 필터링)
- 기간 변경 (5일 기본, 7일 선택 가능)
- 전체 새로고침

**하단 정보**:
- 이번 주 예상 재료비 총액 (실시간)
- 지난주 대비 가격 변동 (트렌드)
- "언제 사면 싼지" 추천

**하단 액션**:
- [메뉴 확정 → 장보기 목록] 버튼

**상태 유지**:
- 메뉴는 LocalStorage에 저장
- 새로고침 버튼 누르면 새로운 메뉴
- 하루가 지나면 자동 리셋

### 4.2 레시피 상세

**진입**: 메인에서 카드 탭/클릭

**구성**:
- Hero 이미지 (고품질)
- 음식명 + 메타 (시간, 난이도, 칼로리)
- 예상 가격 (실시간, 재료별 합산)
- YouTube 영상 링크 → YouTube 앱으로 외부 이동
- 재료 목록 (카테고리별, 각 재료에 실시간 가격)
- 영양정보 (칼로리, 단백질, 탄수화물, 지방)
- 조리순서 (타임라인)
- 요리 팁
- 재뽑기 🎲 버튼

### 4.3 장보기 목록 (보조 도구)

**진입**: 메인에서 "메뉴 확정" 후

**구성**:
- 총 예상 비용 (실시간)
- 진행률 바
- 카테고리별 접이식 섹션
  - 카테고리 소계 가격
  - 재료별: 체크박스 + 이름 + 합산량 + 실시간 가격 + 가격변동(↑↓)
- 목록 복사 버튼

## 5. 가격 모니터링 시스템

### 요구사항
- 재료별 실시간 가격 (쿠팡/마켓컬리/네이버쇼핑 수준의 정확도)
- 재료별 가격 추이 (시계열 — 지난달 대비 변동)
- 총 예상 재료비 (메뉴별 + 주간 합산)
- "언제 사면 싼지" 추천

### 데이터 소스 (리서치 필요)
| 후보 | 장점 | 단점 |
|------|------|------|
| KAMIS (농수산물유통정보) | 공공 API, 무료 | 가공식품 미포함 |
| 쿠팡 스크래핑 | 실제 소비자가 | 법적 리스크, 차단 가능 |
| 마켓컬리 API | 정확한 가격 | 공식 API 없음 |
| 네이버 쇼핑 API | 공식 API 존재 | 최저가 중심 |

**리서치 태스크**: 어떤 소스가 실제로 사용 가능한지 조사 필요

### 가격 데이터 갱신 주기
- 일 1회 배치 업데이트 (권장)
- 또는 요청 시 실시간 조회 (API 비용 고려)

## 6. 이미지 품질 개선

**현재 문제**: 식약처 API 원본 이미지 품질이 낮음

**리서치 필요**:
- 외부 이미지 API (Unsplash/Pexels 음식 카테고리)
- AI 생성 이미지 (DALL-E, Midjourney)
- 식약처 이미지 후처리 (업스케일링)

## 7. 기술적 개선 사항 (리뷰 기반)

### Critical
- [ ] 에러 삼키기 패턴 제거 → toast 알림 도입
- [ ] reroll 실패 시 유저 피드백

### High
- [ ] 클라이언트 상태 관리 도입 (Zustand or Context)
- [ ] `any` 타입 제거 + 타입 안전성 확보
- [ ] API input validation (zod)

### Medium
- [ ] 추천 스코어링 로직 (태그 매칭 가중치)
- [ ] SYNONYMS 확장 (100개+)
- [ ] 재뽑기 시 이전 레시피 재등장 방지

### Low
- [ ] navigator.share API 지원

## 8. 랄프 실행 계획

### Phase 1: 플로우 리디자인 + 버그 수정
- 태그 선택 페이지 제거
- 메인 → 바로 주간 메뉴 (랜덤)
- 필터 UI 추가 (Apple 스타일)
- 에러 핸들링 개선 (toast)
- 상태 관리 도입 (메뉴 유지 + LocalStorage)
- **디자인 리뉴얼** (현대적, 감각적, 애니메이션)

### Phase 2: 가격 모니터링
- 가격 데이터 소스 리서치 + 결정
- 가격 수집 파이프라인 구축
- 재료별 실시간 가격 표시
- 가격 트렌드 (시계열)
- 구매 타이밍 추천

### Phase 3: 품질 개선
- 이미지 품질 개선 (리서치 결과 적용)
- YouTube 외부 링크 방식 변경
- 추천 알고리즘 개선
- SYNONYMS 확장

## 9. 검증 기준

### E2E 검증 (Playwright)
- 앱 오픈 → 5일 메뉴 즉시 표시
- 카드 탭 → 레시피 상세 (재료, 가격, 조리순서)
- 재뽑기 → 새로운 레시피 (이전 레시피 재등장 금지)
- 전체 새로고침 → 완전히 새로운 메뉴셋
- 필터 적용 → 필터에 맞는 메뉴만
- 메뉴 확정 → 장보기 목록 (가격 포함)
- 체크리스트 체크 → 진행률 반영
- 목록 복사 → 클립보드

### 엣지 케이스 테스트
- 필터 조건에 맞는 레시피가 0개일 때
- 재뽑기 10회 연타
- 브라우저 뒤로가기 후 메뉴 유지 확인
- 하루 지난 후 메뉴 리셋 확인
- 네트워크 끊김 상태에서 동작
- 7일로 변경 후 다시 5일로 변경
- 가격 데이터 없는 재료 처리
